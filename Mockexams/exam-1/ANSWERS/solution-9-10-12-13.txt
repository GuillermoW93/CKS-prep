The Combined Solution for Q9, 10, 12, and 13

Here is how you tackle:

(Q9 & Q13 - Require 1): Disabling automount for the ServiceAccount token.

(Q10 & Q12): Container immutability (readOnlyRootFilesystem) and runAsNonRoot.

(Q13 - Require 2): Using a projected volume for the token in a Deployment.

Step 1: Create the ServiceAccount

First, you must create the ServiceAccount and explicitly tell it not to automatically mount its token to pods.

Run this command or apply this YAML:

YAML
apiVersion: v1
kind: ServiceAccount
metadata:
  name: restricted-sa
automountServiceAccountToken: false # Solves Q9 and Q13 (Require 1)
Step 2: Create the Deployment

Next, create the Deployment. This YAML includes the read-only filesystem, the non-root enforcement, the emptyDir workaround for read-only systems, and the projected volume mount.

Apply this deployment.yaml:

YAML
apiVersion: apps/v1
kind: Deployment
metadata:
  name: secure-app-deployment
spec:
  replicas: 1
  selector:
    matchLabels:
      app: secure-app
  template:
    metadata:
      labels:
        app: secure-app
    spec:
      # Attach the custom ServiceAccount
      serviceAccountName: restricted-sa
      
      # Enforce Non-Root at the Pod level (Solves Q12)
      securityContext:
        runAsNonRoot: true
        runAsUser: 1000 # Required so the container doesn't default to UID 0
      
      containers:
      - name: secure-container
        image: busybox:1.36
        command: ["sleep", "3600"]
        
        # Enforce Immutability (Solves Q10 and Q12)
        securityContext:
          readOnlyRootFilesystem: true
          
        volumeMounts:
        # Mount the projected token to the exact path requested (Solves Q13 - Require 2)
        - name: sa-token-projected
          mountPath: /var/run/secrets/kubernetes.io/serviceaccount/token
          readOnly: true
        # Workaround: Apps need to write to /tmp, but the root is read-only
        - name: tmp-volume
          mountPath: /tmp
          
      volumes:
      # Define the Projected Volume for the SA Token
      - name: sa-token-projected
        projected:
          defaultMode: 0444
          sources:
          - serviceAccountToken:
              path: token
              expirationSeconds: 3600 # Standard security best practice
              
      # Define the emptyDir for the /tmp workaround
      - name: tmp-volume
        emptyDir: {}
